Adjustment API

Change Logs
Version Description
v.20220613 Initial Document
v.20221003 Add adjustment_type, adjustment_remarks
Triggers
Upon saving adjustment in FarEye.
Request Parameters
Name Type Required Description
tracking_no String(128) True 2GO Unique reference number. AWB Number of shipment to be updated
adjustment_number String(128) True FarEye generated adjustment number
adjustment_datetime Datetime
(YYYY-MM-DD HH:mm:ss)		 True Date and time when the adjustment happened
adjustment_type String(128) Optional NEW Type of Adjustment
adjustment_remarks String(128) Optional NEW Adjustment remarks
declared_value Decimal(12,2) Optional Updated Declared Value
cod_amount Decimal(12,2) Optional Updated COD Amount
service_type String(128) Optional Updated Service Type
dest_port String(128) Optional Updated Destination Port
dest_area_code String(128) Optional Updated Destination 2GO Area Code
paymode String(128) Optional Updated Paymode. Allowed for CC and CS only
updated_by String(128) True User who made the adjustment
dest_system String(128) True UPDATE Will now be dedicated to ZIP transaction
