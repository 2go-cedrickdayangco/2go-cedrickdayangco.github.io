API Reference for 2GO AR Billing

Use this reference when building AR Billing as these API will be pushed to AR Billing Platform.

Base URL

Base https://api.yourservice.com/api
ℹ
All API requests must be made over HTTPS. Calls made over plain HTTP will be rejected. JSON is the only supported request and response format.

For discussion.

Trigger will be upon receiving of terminating status : DEL, RTS, RTC, WFC

orig_area_code string required Origin Area Code

cnee_name string required Consignee Name

delivery_category string required SA - Serviceable Area
OTD - Out of Town Delivery
NS - Not Serviceable
rc_code string required Revenue Center Code
rc_parent string required Revenue Center Parent : Express, Forwarding
rc_child string required Revenue Center Child Name
business_type string required Business Type
pickup_date date required Pickup Date
reference_number string optional Customer Reference Number
contract_id string optional Contract ID
contract_status string optional Contract Status
item_details array required Wrapper for the list of item details
item_no integer required Item Number
item_desc string required Item Description
item_qty integer required Item Quantity
package_code string required Packaging Code
commodity_code string required Commodity Code

item_height decimal required Item Height in meter
block_measurement string required Block Measurement
rate_id string required Rate Card ID
charges_details array required Charge Details Wrapper
charge_name string required Name of charge
charge_qty integer required Quantity of charge
charge_amount decimal required Total amount of charges

Field Type Required Description

Errors & Responses

The API uses conventional HTTP status codes to indicate the success or failure of requests. Errors return a JSON body with a code and human-readable message.

200 OK — The request succeeded and a resource was returned.
201 Created — A new resource was successfully created.
400 Bad Request — The request was malformed or missing required parameters.
401 Unauthorized — No valid API key provided.
403 Forbidden — The API key doesn't have permission to perform this action.
404 Not Found — The requested resource does not exist.
429 Too Many Requests — Rate limit exceeded. Retry after the period shown in the header.
500 Server Error — Something went wrong on our end. Please retry or contact support.
