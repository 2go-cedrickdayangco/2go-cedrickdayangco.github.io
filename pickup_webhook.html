Request Parameters
Name Type Required Description
tracking_no String True Unique reference string of the transaction. AWB Number
status String True Status Code of the pickup activity. Example : SERV_OK
status_remarks String True New Required if status is FAILED. Reason on why the pickup failed.
Example : Item Not Ready
status_date Datetime
(YYYY-MM-DD HH:ii:ss)		 True Date of when the status was done
status_lat Decimal optional New Latitude of where the status happens
status_lng Decimal optional New Latitude of where the status happens
courier_id String True New Courier ID
courier_name String true New Courier Name
courier_mobile String true New Courier Mobile Number
courier_vehicle_type String true New Courier Vehicle Type. Example : Motorcycle, Bike, Multicab
courier_vehicle_plateno String true New Plate Number of Courier Vehicle

Events and Status Mapping
Status Event Description
TSND_NEW TO SEND - New Booking Newly created booking for pickup.
TSND_RE TO SEND - Rescheduled Booking Rescheduled booking for pickup.
SENT Courier Dispatch The courier is already dispatched for pickup.
SERV_FAIL Failed Pickup due to Seller's Fault The courier fails to pick up the package due to some reason. (e.g. Item is not ready, wrong packaging, seller’s fault, etc..)
USRV Failed Pickup due to Rider's Fault The courier fails to pick up the package due to some reason. (e.g. insuficient time, vehicle capacity, pouch issue, awb issue)
SERV_OK Pickup Successful The courier successfully picks up the package.
